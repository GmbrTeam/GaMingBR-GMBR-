FAQ

Dúvidas frequentes

Não encontrou o que precisava? Fale com a equipe no Discord.

Entrar no Discord
O que é o GMBR Launcher?
O GMBR é um gerenciador de jogos para PC com catálogo de mais de 120.000 títulos, filmes e séries integrados, autenticação completa e ferramentas de diagnóstico. Gratuito, sem anúncios, sem instalação do Python.
O Windows exibiu um aviso de segurança. É seguro?
Sim. O aviso do SmartScreen aparece para qualquer executável sem certificado EV pago. A partir da v4.2 o bundle é assinado digitalmente. Clique em "Mais informações" → "Executar assim mesmo".
Preciso criar uma conta?
Não. O launcher funciona completamente sem conta. A autenticação é opcional — serve para sincronizar favoritos e configurações entre dispositivos.
Como atualizar?
No momento as atualizações são manuais — baixe a versão mais recente no GitHub e instale sobre a existente. O sistema de atualização automática está no roadmap.
O que é o Modo Performance?
Um painel nas configurações com oito toggles que desligam animações, blur e efeitos visuais. O preset "Máximo" ativa todos e sincroniza com o servidor — o cache de respostas sobe de 30 s para 120 s, deixando o launcher mais leve.
O que é o Dados Nerds?
Painel de diagnóstico nas configurações com métricas em tempo real: CPU, RAM, GPU, VRAM, temperaturas, uptime, cache e DOM. Atualizado via /api/nerdstats com intervalo configurável.
Como gerar um log de suporte?
Na aba Suporte do launcher, clique em "Salvar Log — Escolher Pasta". O arquivo gerado contém o diagnóstico completo. Envie-o em um ticket no Discord.
Há versão para macOS?
Não ainda. O macOS está no roadmap como item planejado. Acompanhe as atualizações no Discord.

Regras & Legalização

Uso responsável

Leia com atenção antes de usar o GMBR Launcher. O uso da plataforma implica concordância com estas diretrizes.

Isenção de responsabilidade

O GMBR Launcher é uma ferramenta de organização e indexação. Ele não armazena, distribui nem hospeda nenhum arquivo de jogo. Todo o conteúdo listado é proveniente de fontes públicas de terceiros. A responsabilidade pelo uso é inteiramente do usuário.

Direitos autorais

Todos os jogos, filmes e séries exibidos no catálogo pertencem aos seus respectivos detentores de direitos. O GMBR não reivindica propriedade sobre nenhum desses conteúdos. Caso seja titular de algum direito e queira solicitar remoção, entre em contato via Discord.

Uso pessoal

O GMBR é destinado exclusivamente ao uso pessoal e não comercial. Você pode jogar, assistir filmes, organizar sua biblioteca e personalizar sua experiência livremente, inclusive sem criar conta para a maioria das funcionalidades.

Sem garantias

O software é fornecido "como está", sem garantia de qualquer tipo. Os desenvolvedores não se responsabilizam por danos diretos ou indiretos decorrentes do uso do launcher, incluindo perda de dados ou incompatibilidades de sistema.

Conduta na comunidade

É proibido o uso do GMBR para distribuição de malware, spam, conteúdo ilegal ou qualquer atividade que viole leis locais ou internacionais. Violações resultam em banimento permanente da plataforma e comunidade.

Onde ficam seus dados?

Todos os dados pessoais (biblioteca, histórico, favoritos, configurações) ficam salvos em C:\Users\...\AppData\Roaming\GMBR\. Esses arquivos sobrevivem a reinstalações do launcher.

Dados sensíveis são cifrados com AES-256-GCM. A chave é derivada do identificador único da sua máquina — o banco não pode ser lido em outro computador.

  • Otimizar código para melhor performance
  • Criar equipe de desenvolvimento para colaborar
  • Mudar completamente a aparência visual (cores, layouts, fontes)
  • Compartilhar sua versão modificada gratuitamente
  • Cobrar pelo seu trabalho de desenvolvimento via meios externos

  • Pedir ajuda diretamente ao desenvolvedor oficial
  • Modificar código backend, lógica, integrações, estrutura de arquivos

Você não precisa pedir permissão para modificar — mas precisa informar ao usuário final que é uma versão modificada.

O que você NÃO pode fazer
  • Nome e marca: O app deve continuar sendo chamado "GMBR Launcher". Não pode renomear.
  • Logo: O logo do GMBR não pode ser substituído, removido ou alterado.
  • Código fechado: A versão modificada precisa permanecer open source.
  • Branding oculto: Não esconda o nome "GMBR Launcher" na interface.
  • Monetização interna: Nenhuma loja, pagamento, assinatura ou paywall dentro do app.

Por que essa regra existe?

O desenvolvedor não ganhou nada com o GMBR Launcher — nenhum centavo com o site ou o projeto. Se você criar uma versão paga, estaria ganhando dinheiro com trabalho que não é seu. Além disso, os usuários estariam pagando por algo que deveria ser gratuito, e isso criaria confusão sobre a versão oficial do projeto.

Por que proibir código fechado?

Código fechado abre margem para mineração de dados, spywares e malwares — o usuário não tem como saber o que está rodando. Com código aberto, qualquer pessoa pode auditar e verificar. O GMBR existe porque acredita em software aberto e comunidade: fechar o código mata esse espírito.

Por que proteger nome e logo?

Se alguém renomear o app e fizer coisas ruins, a culpa recairia sobre o GMBR — destruindo a reputação de anos de trabalho honesto. Se você quer sua própria marca, crie um fork completo com nome diferente. É assim que projetos open source funcionam.

Por que exigir transparência para o usuário?

O usuário merece saber exatamente o que está usando. Se ele tiver problemas com uma versão modificada e vier perguntar ao desenvolvedor oficial, é preciso esclarecer que não é a versão oficial. Sem essa transparência, o desenvolvedor original ficaria respondendo por erros de terceiros.

Por que pedir discrição no crédito?

Flood de créditos e assinaturas gigantescas degradam a experiência do usuário e parecem amadoras. Nem o desenvolvedor oficial exibe o nome do projeto de forma agressiva por todo lugar. Existe um equilíbrio entre dar crédito e não incomodar.

Por que tanta liberdade?

O GMBR foi construído para a comunidade. Otimizar, traduzir, criar equipes, mudar a aparência — isso é ótimo e o projeto só ganha com isso. A confiança é a base: você tem liberdade, só precisa respeitar as regras básicas.

Use bom senso. Nem tudo que não foi explicitamente proibido significa que é permitido. Se algo parece errado, provavelmente é. Em caso de dúvida, entre em contato antes de fazer — é melhor perguntar do que violar sem querer.
